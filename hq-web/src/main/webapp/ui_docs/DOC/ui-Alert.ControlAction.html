Hyperic 4.5 Documentation : ui-Alert.ControlAction
This page last changed on Jul 17, 2009 by mmcgarry.

Available only in vFabric Hyperic

You can use the Add Control Action page, which appears when you click Edit on the Control Action tab at the bottom of an Alert Definition page, to specify a control action for HQ to perform when the alert fires, or remove a previously assigned control action from the alert definition.

Feedback is welcome. Click Add Comment at the bottom of the page.

Add a Control Action to an Alert Definition

On the Add Control Action page for an alert definition:

  1. Select the target resource for the control command using the Resource Type and Resource Name pulldowns. The lists allow you to select only a target resource that:
    1. Runs on the same platform as the resource to which the alert definition is assigned, and
    2. Supports one or more HQ control operations, and is configured, as necessary, for control actions.
  2. Select the desired control action from the Control Type pulldown.   
  3. Click .

Note:  You can assign only one control action to an alert definition.

Remove a Control Action From an Alert Definition

On the Add Control Action page for an alert definition:

  1. Select none from Control Type pulldown.   
  2. Click .
Return to editing the alert definition.
