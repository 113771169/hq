Hyperic 4.6 Documentation : Installing an RPM Package

This page has information for Linux administrators who will install Hyperic components from RPM packages. It is assumed that the administrator performing RPM installations is familiar with RPM packages and installation processes.

About Hyperic RPM Packages

The Hyperic Agent in Hyperic RPMs does not include a JRE. Agent hosts must have the J2RE virtual package installed. A Sun 1.5 JRE is recommended.

The Hyperic Server in Hyperic RPMs includes a 1.6 JRE.

The following RPMs are available:

  • vFabric Hyperic RPMs
    • hyperic-hqee-agent-n.n.n.n-n.noarch.rpm — Contains vFabric Hyperic Agent only.
    • hyperic-hqee-installer-n.n.n-n-n.x86_64.rpm — Contains vFabric Hyperic Server for the x86_84 architecture, and vFabric Hyperic Agent.
  • Hyperic HQ RPMs
    • hyperic-agent-n.n.n.n-n.noarch.rpm — Contains Hyperic HQ Agent only.
    • hyperic-hq-installer-n.n.n-n.i386.rpm — Contains Hyperic HQ Server for the x86 architecture, and Hyperic HQ Agent.
    • hyperic-hq-installer-n.n.n-n.x86_64.rpm — Contains the Hyperic HQ Server for x86_64 architecture, and Hyperic HQ Agent.
  • RPM Support Files
    • rpm_support_files-n.n.n.tgz — Support files required for creating your own RPMs for either Hyperic HQ or vFabric Hyperic, including:
    • x86-server.spec — Spec file for hyperic-hq-installer-x.x.n-n.i386.rpm.
    • x86_64-server.spec — Spec file for hyperic-hq-installer-n.n.n-n.x86_64.rpm.
    • noarch-dotorg.spec — Spec file for hyperic-agent-n.n.n.n-n.noarch.rpm.
    • hyperic-hqee-agent.init.rh — Starts and stops the vFabric Hyperic Agent on RedHat Linux.
    • hyperic-hq-server.init.rh — Starts and stops the Hyperic Server on RedHat Linux.
    • hyperic-hq-agent.init.suse — Starts and stops the Hyperic HQ Agent on SUSE Linux.
    • hyperic-hq-agent.init.rh — Starts and stops the Hyperic HQ Agent on RedHat Linux.
    • upgrade.exp — Expect script that upgrades the Hyperic Server, using the Hyperic installer.
    • install.exp — Expect script that installs the Hyperic HQ Server, using the Hyperic installer.
    • install_ee.exp — Expect script that installs the vFabric Hyperic Server, using the Hyperic installer.

Before Installing a Hyperic RPM

  • Configure Hyperic Agent properties — A Hyperic Agent obtains the settings it needs to connect to and communicate with the Hyperic Server at first startup, either interactively, or from startup properties that can be specified in the agent.properties file. If you wish to automate agent installation and configuration, you must edit agent.properties to specify the startup properties. For more information, see Configure Agent - Server Communication in Properties File.
  • Check path to Hyperic Agent JRE — The agent init script is installed as /etc/init.d/hyperic-hq-agent, and is added to the appropriate run-levels via chkconfig. The script assumes the path to your JRE is /usr/java/jdk1.5.0_12. If this is not the case on the target host, modify the init script and specify the path to your JRE in the HQ_JAVA_HOME environment variable.
  • Open firewall port on Hyperic Agent, if necessary — If iptables (a host-based firewall tool typically enabled by default on Redhat and Fedora installations) is configured, you may need to open up the Hyperic Agent's listen port for communication from the Hyperic Server, using a command similar to: 
    `/sbin/iptables -ARH-Firewall-1-INPUT -p tcp --dport 2144 -j ACCEPT`

    If SELinux is enabled, dditional configuration may be required.

  • Start local SMTP server on Hyperic Server host — The RPM installer requires that your SMTP server is listening on port 25 on the Hyperic Server host.

What the RPM Package Does

The RPM package:

  • Creates the "hyperic" user and "hyperic" group if they do not exist.
  • Creates the /opt/hyperic directory is created if it does not already exist. If the "hyperic" user did not exist prior to running the RPM, /opt/hyperic is assigned as its home directory, and appropriate permissions are set.
  • Creates the /opt/hyperic/hq-plugins directory for custom product plugins.
  • Installs the Hyperic Agent in /opt/hyperic-hq-agent.

After Installing the Hyperic Server and Hyperic Agent

If you installed both the Hyperic Server and the Hyperic Agent, start the server first. Start the agent after the server startsup.

  • For information about starting and configuring the server see Configure and Run the Hyperic Server.
  • For information on starting the agent the first time, see Configure and Run the Hyperic Agent.
