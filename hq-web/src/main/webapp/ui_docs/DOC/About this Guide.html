This is an early release of HQU Developer's Guide, a guide to using Hyperic's HQU Framework.
This release of HQU Developer's Guide is based on developer documentation also available on the HQU Forge, on the Hyperic support site. The sections that follow are intended to provide a range of product and usage information, including conceptual, architectural, procedural, reference, and example material.
Some of the content planned for the complete HQU Developer's Guide is not available in this release of the document. Throughout the guide WIP Notes indicate that a section is "work in progress".
Questions, suggestions, and feedback on the document are welcome; such input will form the basis for improvements to make the guide as useful as possible for the new HQU Plugin Developer. Feedback can be posted on the Hyperic HQ User Forum at http://forums.hyperic.com.

Audience

This guide is written for the HQU plugin developerâ€”an HQ user, developer, or partner with a need to extend or tailor HQ. It is assumed that the user is knowledgeable about HQ functionality and features.
Because the HQU Framework is based on the Groovy language, a plugin developer that is not already familiar with Groovy will need to gain basic Groovy competence to use the HQU Framework. Understanding and coming up to speed with Groovy requires a basic understanding of Java and object-oriented design patterns; experience with markup and scripting languages is beneficial. The HQU Developer's Guide is not a guide to Groovy usage.
This document may be of interest to non-developers that are interested in understanding the capabilities of the the HQU Framework or are defining requirements for an HQU Plugin to be developed by someone else.
