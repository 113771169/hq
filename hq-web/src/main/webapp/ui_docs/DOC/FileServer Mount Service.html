FileServer Mount

In Hyperic, a file system is represented by a platform service of type "FileServer Mount". Typically, the Hyperic Agent automatically discovers file systems on a platform. If the agent does not discover the file system on a platform, you can manually configure it.

Create FileServer Mount Service in Hyperic

To create a FileServer Mount service.

  1. Navigate to the platform where the file server exists.
  2. Select New Platform Service from the Tools menu. 
  3. Name the service.
  4. Select "FileServer Mount" from the Service Type pull-down.
  5. Click Edit in the "Configuration Properties" section of the new service's Inventory page.
  6. On the Configuration Properties page, supply at least the required properties, described in the table below.

FileServer Mount Service Configuration Options

Name Description Required Example Notes
path Path to the directory Yes /Users/myfiles Hyperic will verify that the path exists.

Event Tracking

You can enable log tracking for a FileServer Mount service. If you do, the agent will log an event for responses that match the criteria you specify.

To enable log tracking, click the Enable Log Tracking box in the Monitoring section of the Configuration Properties page.

For information about Hyperic log tracking functionality and configuration options, and how to view log events, see Set Up Log Tracking for a Resource in vFabric Hyperic Administration.
