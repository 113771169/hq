vFabric Hyperic 4.6 Features

Note: In addition to the features described in this section vFabric Hyperic 4.6 also includes the features described in Hyperic HQ 4.6 Features.

Enhanced LDAP Support

In vFabric Hyperic 4.6, a Hyperic user can be automatically assigned to Hyperic roles based on groups the user is assinged to in your LDAP system. If your LDAP groups correspond well — in terms of membership — to the roles you will set up in Hyperic, you can use this feature to automate the process of assigning Hyperic roles to a new user.

Given an an LDAP group whose membership is identical to the users you want to assign to a particular Hyperic role, you create a Hyperic role with the same name as the LDAP group.

The first time a user logs on to Hyperic with his LDAP credentials, the user is authenticated via the LDAP system, and register as a Hyperic user. If the LDAP user belongs to any LDAP groups whose names match existing Hyperic roles, those roles are assigned to the new Hyperic user account.

To enable this behavior:

  • When you configure Hyperic Server to use your LDAP system for authentication, you specify the LDAP group search base and associated filter criteria.
  • You create roles with names that match the name of corresponding LDAP groups. role in Hyperic for each LDAP group.

Note that subsequent changes to the LDAP user's group membership are not reflected in the associated Hyperic user's role assignments.

Schedule Maintenance Downtime for a Resource

In vFabric Hyperic 4.6, you can scheduled downtime for an individual resource. In previous releases, downtime could only be scheduled for a resource group. The Schedule Downtime command is available on the Tools menu for a platform, server, service, or group.

Product Plugin for Spring Insight Operations

Spring Insight is a web application that runs in tcServer Developer Edition and analyzes the runtime performance of applications running in the container and provides visibility into it activity on a request-by-request basis. Spring Insight Operations ships as a template with tcServer.

vFabric Hyperic 4.6's Spring Insight plugin collects application metrics from Spring Insight Operations. The plugin provides a user interface that correlates the performance of an application with the performance of the application server in runs in, and also with the platform that hosts the application server. Given a performance problem that appears to be at the application level, you can jump from the Hyperic Insight Applications view into the Insight Operations Dashboard, into the same application context as the problem occurred, to diagnose the underlying problem.

For plugin documentation, see Spring Insight. For information about the Spring Insight view in the Hyperic user interface, see the help page.

Hyperic HQ 4.6 Features

The Hyperic HQ 4.6 features described below are also in vFabric Hyperic 4.6.

Product Plugin Management and Deployment

Hyperic 4.6 provides new features that ease the process of deploying and managing product plugins. These features, described in the paragraphs below, include a user interface for managing product plugins, and new Hyperic Server and Hyperic Agent behaviors for ensuring consistent plugin deployment across a Hyperic environment.

Plugin Manager

Plugin Manager is a new user interface that enables you to deploy and manage plugins across a Hyperic deployment from the Hyperic user interface. You can use Plugin Manager to deploy and remove plugins on the Hyperic Server and all agents reporting to the server.

Server-Agent Plugin Synchronization

The new Server-Agent Plugin Synchronization (SAPS) feature ensures that product plugins are consistently deployed across a Hyperic deployment. SAPS is enabled by new behaviors on the Hyperic Server and Hyperic Agent. When a Hyperic 4.6 Agent starts up, it sends a plugin status report to the Hyperic Server. If an agent's deployed plugins do not exactly match those deployed on the Hyperic server, the server updates the agent's plugins appropriately. Similarly, each time the Hyperic Server is restarted, it compares the product plugins it deploys to most recent plugin status reported by each agent — if an agent's deployed plugins do not exactly match those deployed on the Hyperic server, the server updates the agent's plugins appropriately. SAPS is enabled by default; it can be disabled using the new server.pluginsync.enable server property, described below.

New Hyperic Server Property to Enable and Disable SAPS

Hyperic 4.6 provides a new Hyperic Server property, server.pluginsync.enable, which you can use to disable the SAPS feature described in the previous section. To disable SAPS, add server.pluginsync.enable=false to ServerHome/conf/hq-server.conf, and restart the Hyperic Server.

New Plugin Descriptor Element to Support SAPS

As of Hyperic 4.6, all product plugins must have a version. A new attribute — version — is required in the root <plugin> element in the XML descriptor for a product plugin.

iPad and iPhone Support

The Hyperic 4.6 user interface can be access from iPads and iPhones.

Updated Plugins

All Hyperic plugins have been updated to include the version attribute in the root <plugin> element, required for SAPS.

In addition, the following Hyperic plugins are updated with new functionality, or to support more recent versions of the managed product:

  • GemFire
  • RabbitMQ — In Hyperic 4.6, the RabbitMQ plugin no long requires access the erlang.cookie file — the plugin now uses the rabbitmq-management plugin. The configuration process has changed, and additional metrics are available. For more information see Monitoring RabbitMQ - Hyperic 4.6.
  • tc Server — Support for tc Server 7.0 added.
  • Jboss — Support for Jboss 5.1 and 6.0 added.
  • PostgreSQL — Support for PostgreSQL 9.0 added.
  • WebSphere
  • WebLogic Server Plugin — In Hyperic 4.6, the plugin for managing WebLogic Server is enhanced to enable discovery of node managers and managed servers in a WebLogic domain without accessing the Administration Server for the domain. In previous releases, it was necessary to discover the Administration Server for a domain, and then configure that resource with WebLogic Server credentials, in order to discover the node manager and managed servers in the domain.
