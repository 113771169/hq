WebSphere Plugin Overview

Hyperic's WebSphere plugin manages WebSphere Adminstration Servers and managed WebSphere instances using WebSphere's Performance Monitoring Infrastructure (PMI). The WebSphere supported versions are 6.0, 6.1, and 7.0.

WebSphere Resources and the Hyperic Inventory Model

The table below lists the resource types that the WebSphere plugin manages.

Resource Type Inventory Type Notes
WebSphere Admin server WebSphere Administration Server
WebSphere server A managed instance in the domain managed by the Administration Server.
WebSphere Application service An enterprise application (J2EE application) running in a managed WebSphere instance.
WebSphere Connection Pool service A connection pool in a managed WebSphere instance.
WebSphere EJB service An EJB running in a managed WebSphere instance.
WebSphere Thread Pool service A thread pool in a managed WebSphere instance.
WebSphere Webapp service A web application running in a managed WebSphere instance.

The resource hierarchy is:

Requirements

These requirements apply to WebSphere Administration Servers and to managed WebSphere instances.

  • JDK — The Hyperic Agent that manages a WebSphere instance use the WebSphere JDK.
  • SSL — If a WebSphere 6.0 or 6.1 instance has Global Security is enabled:
    • the user running the agent must have has read access to WASinstallpath/properties/ssl.client.props
    • The following properties in that file must be specified correctly: 
      com.ibm.ssl.keyStore
com.ibm.ssl.keyStorePassword
com.ibm.ssl.trustStore
com.ibm.ssl.trustStorePassword

Configure Monitoring

Perform these steps on each platform with WebSphere instances to be monitored.

  1. Configure the Hyperic Agent to use the WebSphere JDK.
    • Define the HQ_JAVA_HOME variable and set it to the Java directory in the WebSphere installation.
      • On Unix-like platforms, set HQ_JAVA_HOME as an environment variable. 
        % export HQ_JAVA_HOME=${was.install.root}/java
      • On Windows platforms, set HQ_JAVA_HOME as a system variable. 
        My Computer > Properties > Advanced > Environment Variables > System variables > New
  2. Install Hyperic Agent.
    • If WebSphere Global Security is defined, install the agent under an account that has read-access to WASinstallpath}/properties/ssl.client.props.
    • For instructions, see Install an Agent-Only Package in Getting Started with vFabric Hyperic.
  3. Configure WebSphere-related agent properties in AgentHome/conf/agent.properties.
    • websphere.installpath — Use this property to define WebSphere installation path. Does this override the default value of was.install.root?
    • websphere.useext — For WebSphere 6.0 or 6.1 you must set this property to "true".
      For information about configuring agent properties, see Set Up Agent in Properties File.
