Viewing and Editing a Role

Available only in HQ Enterprise

This screen allows users to view and manage an individual role. This screen displays the permissions set for the role, the groups of resources assigned to the role, the users assigned to the role, and the calendar detailing when users assigned to this role can be notified about an alert.

Tasks Available on This Screen

On this screen, users can do (or at least start) the following tasks:

To change the role's properties or permissions:

  • In the "Properties & Permissions" section, click and proceed to the "Edit" screen.

To assign a user to the role:

  • In the "Assigned Users" section, click and proceed to the "Add Users to Roles" screen.

To remove a user from the role:

  • In the "Assigned Users" section, check the user and click .

To assign a group to the role:

  • In the "Assigned Groups" section, click and proceed to the "Add Groups to Roles" screen.

To remove a group from the role:

  • In the "Assigned Groups" section, check the group and click .

To change the alert calendar for this role:

  1. In the "Alert Calendar" section, check the days and specify the times (and exceptions) when users assigned to this role can be notified about alerts.
  2. Click Save.

Sections on This Screen

Properties & Permissions

This section lists basic information about the role:

  • Name
  • Owner
  • Description
  • Administer HQ Server Configuration: Indicates whether the users in this role may configure the HQ Server Settings. This is NO by default.
  • Dashboard Name: The name of the dashboard view associated with this role. Learn more about role-based dashboards.

Assigned Users

This section lists the users assigned to this role. To view a user listed here, click the user's name. Users can add and remove users from the role.

Assigned Groups

This section lists the groups assigned to this role. To view a group listed here, click the group's name. Users can add and remove groups from the role.

The Final Step
Even if users are assigned to the role, the role's (and users') permissions will not apply to any specific resources until a group (of said specific resources) is also assigned to the role.

Alert Calendar

This section displays the calendar that dictates when members of this role can be contacted.

Navigating via the Masthead Menu

Main screens in Hyperic contain a masthead menu. Users can navigate to the following parts of HQ using this menu:

Dashboard Displays the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Resources Click the menu name to access either Browse (takes users to the Browse Resources screen, where they can locate and navigate through information about all managed resources), Currently Down (takes users to the Currently Down screen, where they can look at all managed resources that are unavailable), Nagios Availability (available only when Nagios is installed), or Recently Viewed (provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the Current Health screen for the resource).
Analyze Click the menu name to access Reporting * (presents the Reporting screen, from which reports can be generated), Alert Center (presents the Alert Center, which displays a deployment wide alert summary), or Event Center (presents the Event Center, which displays a deployment-wide event summary).
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the Administration screen, where they can manage the system-wide settings
Search
The Search box in the upper right of the Masthead allows you to search for HQ resources of any type and HQ users.

Search results appear after four characters are entered in the text box. Results will include the first 10 platforms, servers, services, and groups whose name includes the search string.

Double-click a resource in the list to navigate to its Resource page.

The masthead also displays the two most recently triggered alerts. Users can click the alert's time to be taken to the alert detail.
