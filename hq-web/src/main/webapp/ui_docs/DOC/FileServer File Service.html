FileServer File

In Hyperic, a file in a file system is represented by a platform service of type "FileServer File". Although a Hyperic Agent automatically discovers filesystems on a platform (see FileServer Mount Service, to monitor a particularly file, you must configure it manually.

Create FileServer File Service in Hyperic

To create a FileServer File service.

  1. Navigate to the platform where the file exists.
  2. Select New Platform Service from the Tools menu. 
  3. Name the service.
  4. Select "FileServer file" from the Service Type pull-down.
  5. Click Edit in the "Configuration Properties" section of the new service's Inventory page.
  6. On the Configuration Properties page, supply at least the required properties, described in the table below.

File Server File Service Configuration Options

Name Description Required Example Notes
path Path to the file Yes /Users/myfiles HQ will check that the path exists

Event Tracking

You can enable log tracking for a FileServer File service. If you do, the agent will log an event for responses that match the criteria you specify.

To enable log tracking, click the Enable Log Tracking box in the Monitoring section of the Configuration Properties page.

For information about Hyperic log tracking functionality and configuration options, and how to view log events, see Set Up Log Tracking for a Resource.
