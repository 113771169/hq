Viewing a Resource's Metrics

Topics marked with * relate to features available only in vFabric Hyperic.

This screen displays all the metrics, in numeric form, that HQ collects for the resource in the metric display range, even those metrics for which no data has been collected. The metrics displayed on the screen vary by resource type; find out which metrics can be collected for this resource in the list of managed products.

Tasks Available on This Screen

On this screen, users can do the following:

To change the display of metrics:

Desired Display Instructions
Only metrics with data Click next to Hide Metrics Without Data at the top of the tab.
All metrics collected for the resource Click next to Show All Metrics at the top of the tab.
Specific metric categories Check or uncheck categories on the left side of the screen and click .
Metrics with specific trends Check or uncheck the "Trends Up" and "Trends Down" on the left side (under "Value Types") and click .
Only Dynamic or only Static metrics Check or uncheck "Dynamic" and "Static" on the left side (under "Value Types")and click .

The display automatically refreshes.

To change the frequency with which the screen refreshes:

  • In the upper-right corner of the tab, to the right of Metrics Refresh, click a refresh value or, to make the display static, click OFF.

To start collecting data for a metric:

  1. In the ""Show All Metrics" view, check the desired metrics (that is, the metrics that HQ is not collecting data for that it should collect data for).
  2. At screen bottom, to the right of Collection Interval for Selected, type the frequency (interval) with which HQ should collect this metric's data and click .
    HQ will now collect data for this metric at the specified interval.

To stop collecting data for a metric:

  • In the ""Show All Metrics" view, check the desired metrics and, at screen bottom, click .
    For groups, this stops the collection for all member resources.

To change a metric's collection interval:

  1. Check the metric.
  2. At screen bottom, to the right of Collection Interval for Selected, type the new collection interval for the click .
    For grouoss, this changes the collection interval for all member resources.

To change the metric display range:

  • Follow the instructions for doing the same thing on the "Current Health" screen.

To display a secondary window with a single metric's details:

  • Click in a metric's row.

For Single Resources

To set a metric's acceptable low, acceptable high, and baseline values:

  • Check the metrics and click .
    This sets those baseline values of the selected metrics to the displayed LOW, PEAK, and AVG values, respectively.

To chart a metric:

  • Check the metrics and click and proceed to the "Metric Chart" screen.

For Group Resources

To compare metrics among members of the group:

  • Check the member resources in "Current Health of group's <group name> resources collecting metrics" and click and proceed to the "Compare Metrics" screen.

To chart a metric for 10 members of the group:

  • Click the metric name and proceed to the "Metric Chart" screen.
    Metric data for the top 10 group resources (as listed in "Current Health of group's <group name> resources collecting metrics"), be default, will be displayed in this chart. On the "Metric Chart" screen this set of resources can be changed to include others from the group.

To view the current health of a member resource:

  • Click the resource's name in "Current Health of group's <group name> resources collecting metrics").

Sections on This Screen

Resources Tab

The tab primarily allows users to filter the list of metrics displayed at the right. It also displays the same "hosted resources" and "hosting resources" portlets as displayed in the "Current Health" screen.

Metric Data Tab

Each metric is displayed with its low, average, peak, and last-recorded values and its collection interval. Also displayed is the number of times a collected metric value triggered an alert ("Alerts") and how many times it was out-of-bounds ("OOB"). Again, all these values are selected only from the metric display range.

The value of a metric's Collection Interval indicates whether or not data is being collected for the metric. A value of "NONE" indicates that data is not being collected. Any other value indicates the frequency with which HQ collects the data.

Metrics can be charted and also their baseline values can be set.

Metric Display Range

This section displays the metric display range used in this, and all Monitoring, screens. Users can change the display range.

Group Metrics

This screen displays summary metric data for groups (both user-created compatible groups and HQ-created autogroups). For compatible groups, HQ displays metric data much like it does for a single resource, but it aggregates metrics at the group level and also displays the SUM. The screen indicates the status of metric collection for the group's member resources with data such as # Collected (how many member resources have collected data for the listed metric). Users can compare all listed metrics for a selected set of member resources, and they can select a metric to be charted for 10 of the group's members.

In the case of an autogroup, this screen displays all metrics available for collection for the autogroup's resource type. The "Members Collecting" section denotes if any member of the autogroup is collecting that metric: "NO" (none), "YES" (all), or "SOME." If a metric is currently being collected, its Collection Interval is also listed: either a specific value if all members that are collecting the metric have the same collection interval, or "DIFFERENT" if the values vary.

Changing values (stopping collection, starting collection, collection interval, etc.) for a metric in a group imposes that one value on all the group's members.

Navigating via the Masthead Menu

Main screens in Hyperic contain a masthead menu. Users can navigate to the following parts of HQ using this menu:

Dashboard Displays the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Resources Click the menu name to access either Browse (takes users to the Browse Resources screen, where they can locate and navigate through information about all managed resources), Currently Down (takes users to the Currently Down screen, where they can look at all managed resources that are unavailable), Nagios Availability (available only when Nagios is installed), or Recently Viewed (provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the Current Health screen for the resource).
Analyze Click the menu name to access Reporting * (presents the Reporting screen, from which reports can be generated), Alert Center (presents the Alert Center, which displays a deployment wide alert summary), or Event Center (presents the Event Center, which displays a deployment-wide event summary).
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the Administration screen, where they can manage the system-wide settings
Search
The Search box in the upper right of the Masthead allows you to search for HQ resources of any type and HQ users.

Search results appear after four characters are entered in the text box. Results will include the first 10 platforms, servers, services, and groups whose name includes the search string.

Double-click a resource in the list to navigate to its Resource page.

The masthead also displays the two most recently triggered alerts. Users can click the alert's time to be taken to the alert detail.
