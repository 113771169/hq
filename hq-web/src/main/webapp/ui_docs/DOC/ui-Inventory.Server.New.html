When to Create a Server Manually

Most server types that HQ manages are auto-discovered, rather than manually added to inventory on the New Server page.

You may need to create a server manually under some circumstances, for instance, if:

  • The plugin that manages it does not support server auto-discovery for the server type.
  • HQ did not discover the server because it is installed in a non-standard location, not in the the location where HQ looks for it.
  • The server instance has a non-standard name (one that the plugin does not recognize) in the process list.
Need to Know What a "Server" Is?
See HQ Inventory Model for a more information.

Add a Server to Inventory from the HQ User Interface

To create a new server:

  1. Use Browse > Resources > Platforms to navigate to the platform to which you wish to add a server.
  2. Select New Server from the Tools menu.
  3. On the New Server page enter:
    • Name: The name of the server
    • Description: (optional) A description of the server
    • Server Type: Select the server type for the new resource.
    • Install Path: The full path of the directory where the server is installed.
  4. Click OK.
    • The Inventory tab for the resource appears.
    • The server's Owner defaults to the account that created the server.
    • The Host Platform is the name of the platform to which you add the server.
  5. If the resource must be configured to enable monitoring, click Edit in the Configuration Properties section of Inventory tab.
