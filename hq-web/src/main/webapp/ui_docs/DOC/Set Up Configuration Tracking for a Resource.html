Hyperic Configuration Tracking Functionality

For supported resource types, you can configure Hyperic to track changes made to configuration files (or other local files of interest) and log events for the associated resource. For a full description of configuration tracking functionality, see Log and Configuration Event Tracking.

Note that:

  • The Hyperic Agent must be able to read a file to track it — ensure that file permissions are such that the Hyperic Agent can read files you wish to track.
  • Configuration tracking is supported for most platform and server types; typically not for services.
  • Configuration tracking functionality for server types varies by resource type, plugins that have been updated to use the org.hyperic.hq.product.FileChangeTrackPlugin added in Hyperic 4.6.5 provide more detailed tracking than those that still use org.hyperic.hq.product.ConfigFileTrackPlugin.

Set Up 4.6.5 Configuration Tracking for a Server

Hyperic 4.6.5 has a new configuration tracking support class that enables enhanced configuration tracking for servers. The new class — FileChangeTrackPlugin ---- supports detailed change tracking of files in multiple directories (or directory trees). The plugin tracks the nature of a change (whether it was an file add, delete, modify, or rename), and retains the "diff" data for the change. In Hyperic 4.6.5, a number of plugins have been updated to use FileChangeTrackPlugin — follow the instructions in this section to set up configuration tracking for servers of these types.
  • Tomcat
  • Apache
  • WebSphere
  • WebLogic Server
  • JBoss
  • IIS
  • PostgreSQL
  • mySQL

To enable and configure configuration tracking for a server:

  1. Navigate to the server.
  2. Select Configure Server from the Tools menu.
    • The Configuration Properties page appears.
  3. In the Monitoring section of the page:
    1. Make sure that the Enable Config Tracking box, which enables and disables configuration tracking for the server, is checked.
    2. In the "Configuration Files" box, specify the files you wish to monitor in this fashion: 
      path;recurse;regex1|regex2,path2;recurse;regex1|regex2|...|regexN
      • where:
        • path — An absolute path, or a path relative to the managed resource's discovered install directory.
        • recurse — Indicates, with a value of "true" or "false", whether or not track the files inside directories below the target directory specified by path.
        • regex — A regular expression for filtering the directories or files to track for changes, in the directory (or directory tree for recursive tracking) specified by path.
          You do not need to configure the files to track, if you are satisfied by the default filter definitions, which are described below.
    3. Click OK.

Tomcat Configuration Tracking

For Tomcat the default value of the Configuration Files field, which specifies which files to track is:

conf;true;.*.properties|.*.xml|.*.policy;,bin;false;.*.bat|.*.xml|.*.sh;,
lib;false;.*.jar;,webapps;true;.*.properties|.*.xml|.*\.jar|.*\.dll|.*\.class
|.*\.jsp|.*\.php|.*\.pl|.*\.js|.*\.py|.*\.pyc|.*\.cgi;

When configuration tracking is enabled, this filter causes Hyperic to track:

  • In the TomcatHome/conf directory and any directories below it, any files that end with the string xml, properties, or policy.
  • In the TomcatHome/bin directory, any files that end with the string bat, xml, or sh.
  • In the TomcatHome/lib directory, any files that end with the string jar.
  • In the TomcatHome/webapps directory and any directories below it, any files that end with the string properties or xml, or have the .jar, .dll, .class, .jsp, .phpl, .pl, .js, .py, .pyc, or .cgi extension.

Apache Configuration Tracking

For Apache, the default value of the Configuration Files field, which specifies which files to track is:

httpd/conf;false;.*\.conf;,httpd/conf.d;false;.*\.conf;,httpd/conf;false;
.*.conf;,httpd/conf.d;false;.*.conf;

When configuration tracking is enabled, this filter causes Hyperic to track files whose name ends in conf or that have the .conf extension in the following directories:

  • ApacheHomehttpd/conf
  • ApacheHomehttpd/conf.d

WebSphere Configuration Tracking

For WebSphere, the default value of the Configuration Files field, which specifies which files to track is:

installedApps;true;.*\.jar|.*\.dll|.*\.class|.*\.jsp|.*\.php|.*\.pl|.*\.vb|
.*\.vbs|.*\.js|.*\.py|.*\.pyc|.*\.cgi;,config;true;.*\.xml|.*\.cfg|.*\.properties|
.*\.ini|.*\.conf|.*\.config;,properties;true;.*\.xml|.*\.cfg|.*\.properties|
.*\.ini|.*\.conf|.*\.config|.*\.policy|.*\.props|.*\.key;

When configuration tracking is enabled, this filter causes Hyperic to track:

  • In the WebSphereHome\installedApps directory and any directories below it, any files with one of the following extensions: .jar, .dll, .class, .jsp, .php, .pl, .vb, .vbs, .js, .py, .pyc, .cgi
  • In the WebSphereHome\config directory and any directories below it, any files with one of the following extensions: .xml, .cfg, .properties, .ini, .conf, .config
  • In the WebSphereHome\properties directory and any directories below it, any files with one of the following extensions: .xml, .cfg, .properties, .ini, .conf, .config, .policy, .props, .key

.*\.cgi|.*\.so.*\.xml|.*\.cfg|.*\.properties|.*\.ini|.*\.conf|.*\.config|.*\.ldift|

JBoss Configuration Tracking

For JBoss, the default value of the Configuration Files field, which specifies which files to track is:

server/all/conf;true;.*\.xml|.*\.cfg|.*\.properties|.*\.ini|.*\.conf|.*\.config;,
server/default/conf;true;.*\.xml|.*\.cfg|.*\.properties|.*\.ini|.*\.conf|
.*\.config;,server/default/deploy;true;.*\.jar|.*\.dll|.*\.class|.*\.jsp|.*\.asp|
.*\.aspx|.*\.php|.*\.pl|.*\.vb|.*\.vbs|.*\.js|.*\.py|.*\.pyc|.*\.cgi|.*\.xml|.*\.cfg|
.*\.properties|.*\.ini|.*\.conf|.*\.config;,server/minimal/conf;true;.*\.xml|.*\.cfg|
.*\.properties|.*\.ini|.*\.conf|.*\.config;,server/standard/conf;true;.*\.xml|
.*\.cfg|.*\.properties|.*\.ini|.*\.conf|.*\.config;,server/web/conf;true;.*\.xml|
.*\.cfg|.*\.properties|.*\.ini|.*\.conf|.*\.config;,;false;jar-versions.xml;,
configuration/;false;.*\.xml|.*\.properties

When configuration tracking is enabled, this filter causes Hyperic to track files:

  • In any directory in the JBossHome/server/all/conf directory and any directories below it, files with the following extensions: .xml, .cfg, .properties, .ini, .conf, .config
  • In any directory in the JBossHome/server/default/conf directory and any directories below it, files with the following extensions: .xml, .cfg, .properties, .ini, .conf, .config
  • In any directory in the JBossHome/server/default/deploy directory and any directories below it, files with the following extensions: .jar, .dll, .class, .jsp, .asp. .aspx, .php, .pl, .vb, .vbs, .js, .py, .pyc, .cgi, .xml, .cfg, .properties, .ini, .conf, .config
  • In any directory in the JBossHome/server/minimal/conf directory and any directories below it, files with the following extensions: .xml, .cfg, .properties, .ini, .conf, .config
  • In any directory in the JBossHome/server/standard/conf directory and any directories below it, files with the following extensions: .xml, .cfg, .properties, .ini, .conf, .config
  • In any directory in the JBossHome/server/web/conf directory and any directories below it, files with the following extensions: .xml, .cfg, .properties, .ini, .conf, .config
  • In the JBossHome directory, the file jar-versions.xml
  • In the JBossHome/configuration/ directory, files with the following extensions: .xml, .properties;

IIS Configuration Tracking

For JBoss, the default value of the Configuration Files field, which specifies which files to track is:

C:\Windows\system32\inetsrv;false;metabase.xml,wwwroot;true;.*.xml|.*.jar|.*.dll|
.*.aspx|.*.asp|.*.js|.*.vbs|.*.ini|.*.conf|.*.properties

When configuration tracking is enabled, this filter causes Hyperic to track:

  • In the C:\Windows\system32\inetsrv directory, the metabase.xml file
  • In wwwroot and any directories below it, any files that end in one of the following strings:  xml, jar, dll,aspx, asp, js, vbs, ini, conf, properties.

PostgreSQL Configuration Tracking

For PostgreSQL, the default value of the Configuration Files field, which specifies which files to track is:

data;false;.*\.conf;,*PGDATA*;false;.*\.conf;,data;false;.*\.conf;

When configuration tracking is enabled, this filter causes Hyperic to track:

  • In the PostgresqlHome/data directory, any files with a .conf extension.
  • In the configuration directory specified by the PGDATA environment variable, and any files with a .conf extension.
  • In the PostgresqlHome/data directory, any files with a .conf extension.

mySQL Configuration Tracking

For MySQL, the default value of the Configuration Files field, which specifies which files to track is:

/etc;false;my\.cnf;,/etc/mysql;false;my\.cnf;,;false;my\.cnf;,~/;false;
\.my\.cnf;,/usr/local/mysql/data;false;my\.cnf;,/usr/local/var;false;my\.cnf
;,%WINDIR%;false;my\.ini|my\.cnf;,c:;false;my\.ini|my\.cnf;,;false;my\.ini|my\.cnf;

When configuration tracking is enabled, this filter causes Hyperic to track my.cnf in the following directories:

  • MySqlHome/etc
  • MySqlHome/etc/mysql
  • MySqlHome
  • the home directory of the user under which MySQL runs
  • /usr/local/mysql/data
  • /usr/local/var
  • the Windows directory
  • c:/

and track my.ini in the following directories:

  • the Windows directory
  • c:/
  • MySqlHome

Set Up Polling-Based Configuration Tracking

For platforms and services that support configuration tracking, and server types that have the agent uses a cryptographic hash function to continuously compare a original version of the file with the current version to see if it has changed.

To enable and configure configuration tracking for a platform or service:

  1. Navigate to the resource instance.
  2. Select Configure ... from the Tools menu.
    • The Configuration Properties page appears.
  3. On the Edit Configuration Properties page:
    1. Click the Enable Config Tracking box to turn configuration tracking on and off for the resource.
    2. Enter the path, relative to the resource's installation directory, to each file you wish to track. Separate files with a comma.
    3. Click OK.

View Configuration Events for a Resource

You can view configuration event data on:

  • The Monitor page for a resource. For more information, see ui-Monitor.CurrentHealth in vFabric Hyperic User Interface.
  • The Event Center page. For more information see ui-Event.Center in vFabric Hyperic User Interface.

  1. Enable and configure configuration tracking as described above.
  2. For the instructions in Define an Alert for a Resource, or Define an Alert for a Resource Type and choose the "Config Changed" condition type.
