Adding a Platform to the HQ Inventory

Topics marked with * relate to HQ Enterprise-only features.

This screen allows users to create ("add") a new platform to the HQ inventory. Users might manually add a platform (or service or server) under the following circumstances:

  • It is a technology that is not monitored by default by HQ.
  • It is a custom plugin.
  • It is installed in a nonstandard location that HQ does not look in by default.
  • It has a nonstandard name in the process list.
Platforms That Cannot be Manually Created
Users cannot create platforms of a type that HQ can auto-discover. These types include Linux, Windows, Solaris, HPUX, Apple OS X, FreeBSD, and AIX. Platforms of these types are added to the HQ inventory by importing them in the Auto-discovery portlet on the Dashboard.

Users can manually create platforms of the following types:

  • Google AdWords (instructions)
  • Cisco IOS (instructions)
  • Network Device (instructions)
  • Network Host(instructions)

Tasks Available on This Screen

On this screen users can do one thing: add a new platform.

To add a new platform:

  1. Fill in the following fields:
    • Name: The name of the platform
    • Description: (optional) A description of the platform
    • Location: (optional) The physical location of the platform hardware
    • Platform Type: The type of platform (This value cannot be changed.)
    • Fully Qualified Domain Name: The platform's FQDN
    • Agent Connection: The IP address:port pair of the Agent the proxied connection to this platform should go through. When a platform is created manually (on this screen), the platform cannot or does not run an Agent itself, and therefore the Agent must connect to the platform via proxy. The list contains the IP address:port pairs from all the currently deployed Agents.
    • IP Address, (optional) MAC Address, (optional) Netmask: At a minimum specify values for the loopback (local) IP address, and then additional sets of values for each network interface on the device.
  2. Click .
  3. Follow the platform-type specific instructions here:
    • Google AdWords
    • Cisco IOS
    • Network Device
    • Network Host

The platform's Owner defaults to the account that created the platform.
