Viewing Service Inventory

Available in HQ Open Source unless marked by * for HQ Enterprise only

This inventory screen allows users to view and manage a selected service. The screen displays a service's user-specified properties, the platformthat hosts the service, the applications containing this service, groups containing the service, and the service's monitoring and control configuration. The service's name is at the rightmost of the path listed at the top of the screen.

Not all screen elements described in this wiki page are available for all instances of a service. Some service types simply have more or fewer data elements associated with them.

Tasks Available on This Screen

On this screen, users can do (or at least start) the following tasks:

To view a map representation of the service in relationship to the server and platform that host it:

  • Click in the upper left.
    The service is displayed at the top, the server that hosts it just below, and the platform that hosts the server just below that. All resource names are bolded and the resource type is displayed just below in smaller type. From there you can navigate to the platform or server by clicking its name.

To change the service's owner:

  • In the middle near the top of the screen, click Change next to the listed owner and proceed to the "Edit" screen.

To change the service's name and description:

  • In the "General Properties" section, click the top and proceed to the "Edit" screen.

To add this service to the user's Dashboard favorites:

  • In the Tools Menu in the upper right, select Add to Dashboard Favorites.

To change the install path of the service:

  • In the "General Properties" section, click the bottom and proceed to the "Edit" screen.

To add this service to a group:

  • In "Groups containing this resource," click and proceed to the "Add to Groups" screen.

To remove this service from a group:

  • In "Groups containing this resource," check the group and click .

To change the any of the server's configuration (log- and configuration-tracking, control actions, etc.):

  • In "Configuration Properties," click and proceed to the "Edit" screen.

Sections on This Screen

Top Level

The top of the page displays high-level identifying information for the service being viewed. The data fields displayed depend on the service type.

General Properties

This section lists the properties the user specified when creating or last modifying the server:

  • Host Platform: The platform that hosts this service
  • Description: A description of the service
  • Resource type: The service type
  • Date Created: The date the service was added to the HQ inventory
  • Date Modified, Modified By: The date the service properties were last modified and the user who modified them

Applications

This section lists the applications of which this service is a member. To view an application listed here, click the application's name.

Groups Containing This Resource

This section lists the groups of which this service is a member. To view a group listed here, click the group's name.

Configuration Properties

This section displays the shared, monitoring, and control configuration properties of the service.

HQ often provides reasonable defaults for the configuration properties, but HQ users must provide information specific to their environment.

Navigating via the Masthead Menu

Main screens in Hyperic contain a masthead menu. Users can navigate to the following parts of HQ using this menu:

Dashboard Takes users to the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Resources Click the menu name to access either Browse (takes users to the "Browse Resources" screen, where they can locate and navigate through information about all managed resources), Currently Down (takes users to the "Currently Down" screen, where they can look at all managed resources that are unavailable), Nagios Availability (available only when Nagios is installed), or Recently Viewed (provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the "Current Health" screen for the resource).
Analyze Click the menu name to access either Reporting * (takes users to the "Reporting" screen, where they can generate all reports available in HQ), Alert Center (takes users to the Alert Center, where they can look at all alerts triggered across the system).
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the "Administration" screen, where they can manage the system-wide settings

The masthead also displays the two most recently triggered alerts. Users can click the alert's time to be taken to the alert detail.
