Cloning a Platform*

Available in HQ Open Source unless marked by * for HQ Enterprise only

This screen allows users to copy the inventory resources, such as server configuration properties and services such as HTTP checks or pings, on one platform to one or more other platforms that exists in HQ inventory.  The cloning process:

  • Clones all servers on the source platform to the target platforms.
  • Does not clone the services that comprise the cloned servers; the child services will be added through the auto-discovery process.
  • Clones all manually created platform services on the source platform to the target platforms.
  • Does not clone auto-discovered platform services on the source platform to the target platforms.

The Clone Platform page is available on the Tools menu when a platform is selected.

If you have any comments or suggestions for this help page, please submit them at the bottom of the page by clicking Add Comment.

To clone a platform:

  1. Navigate to the platform whose inventory resources you wish to clone.
  2. Choose Clone Platform from the Tools menu. 
  3. On the Clone Platform page, the Available clone targets list presents a list of all platforms in inventory. You can narrow the list by entering a string in the Search resources box.
  4. Move the target platforms that you wish to configure from the Available clone targets list to the Selected clone targets list.  
  5. Click Queue for Cloning. Cloning occurs asynchronously. You can perform other Portal functions without waiting for the cloning process to complete.
