Alert Center

Available in Open Source unless marked by * (Available in Enterprise Edition)

This screen allows users to view all the triggered alerts across the entire HQ deployment, sorted by date and time, in backwards chronological order. Users can view either regular alerts or group* alerts.

Tasks Available on This Screen

On this screen, users can do the following tasks:

To view regular or group* alerts:

  • Select "Alerts" or "Group Alerts" in Select alert type.

To view details of a triggered alert:

  • Click the alert name in the "Alert" column.

To view a specific resource that has an alert:

  • Click the resource's name in the "Resource" column.

To sort the list of alerts:

  • Click any column heading.
    The sort will apply to all alert data from the database, not just the alerts currently displayed on the screen.

Sections on This Screen

For each listed alert, the screen lists:
Date: The date and time the alert was triggered
Alert: The name of the triggered alert
Resource: The resource the alert was triggered on
Fixed: Whether or not the alert was fixed
Acked by: The name of the user who acknowledged the alert
Priority: The alert's priority

All alerts that have no yet been fixed are highlighted in pink.

Navigating via the Masthead Menu

Main screens in Hyperic contain the same menu in the masthead. Users can navigate to the following parts of HQ using this menu:

Dashboard Takes users to the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Browse Resources Takes users to the screen where they can locate and navigate through information about all managed resources
Alert Center Takes users to the Alert Center, where they can look at all alerts triggered across the system
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the Administration screen, where they can manage the system-wide settings
Recent Resources Provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the Current Health screen for the resource.
Help Opens screen-sensitive online help in a secondary window
