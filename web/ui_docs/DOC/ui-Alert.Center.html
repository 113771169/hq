Alert Center

Topics marked with * relate to HQ Enterprise-only features.

This screen allows users to centrally view all the triggered alerts and all the alert definitions across the entire HQ deployment. Users can view single-resource, resource-type*, or group * alerts and alert definitions. The screen automatically refreshes every three minutes, but users can also refresh it manually.

Tasks Available on This Screen

On this screen, users can do the following tasks:

To refresh the list:

  • Click at the top right.

To sort the list:

  • Click any column heading in the list at right.

When viewing triggered alerts, the sort will apply to all alert data from the database, not just the alerts currently displayed on the screen.

As of HQ 4.1, you can acknowledge and fix alerts on the Alert Center page:

To acknowledge alerts

  • Check-mark the alert(s) you wish to acknowledge and click Acknowledge. As of HQ 4.1.1, if the escalation has pause options, you are offered the option to pause the escalation.

To fix alerts

  • Check-mark the alert(s) you wish to fix and click Fixed. A pop-up will prompt you to enter a comment about the fix. As of HQ 4.1.1, if there are other unfixed alerts with the same definition, you are offered the option to "Fix all previous alerts".

Alerts Tab

To view alerts for single resources or groups*:

  • On the "Alerts" tab, select "Resource" (alerts triggered on an individual resource, from either a resource-specific alert definition or a resource-type alert definition) or "Group" (alerts triggered on a group) in Alert Type.

The list at the right is automatically filtered.

To filter the list of displayed alerts:

  • On the "Alerts" tab, select a filter value in "Alert Filter":
    • Minimum Priority: The minimum priority the alert must have
    • In the last: The length of time, dating back from now, in which to search for triggered alerts

The list at the right is automatically filtered.

To view details of a triggered alert:

  • On the "Alerts" tab, click the alert name in the "Alert Definition" column.

To view the resource for which an alert was triggered:

  • On the "Alerts" tab, click the resource's name in the "Resource" column.

Definitions Tab

To view alert definitions for single resources, resource types**, or groups*:*

  • On the "Definitions" tab, select "Resource," "Resource Type," or "Group" in Definition Type.

The list at the right is automatically filtered.

To view only those alert definitions that are disabled:

  • On the "Definitions" tab, check Only show disabled definitions.

The list at the right is automatically filtered.

To exclude resource-type alert definitions:

  • On the "Definitions" tab, check Exclude type-based definitions.

The list at the right is automatically filtered.

To view details of the last alert triggered from this alert definition:

  • On the "Definitions" tab, click the date and time in the "Last Alert" column.

Sections on This Screen

Alerts Tab

This section lists all the alerts (resource alerts and group alerts) triggered across the HQ deployment, in a flat hierarchy, sorted by triggered date and time, in backwards chronological order. (Alerts that get generated from resource-type alert definitions are considered resource alerts.) In the "Alert Filter" section, users can filter the list of triggered alerts displayed at the right.

For each listed alert, the screen lists:

  • Date: The date and time the alert was triggered
  • Alert Definition: The name of the alert definition that prompted the alert
  • Resource: The resource the alert was triggered on
  • Fixed: Whether or not the alert was fixed
  • Acked by: The name of the user who acknowledged the alert
  • Priority: The alert's priority

Alerts that have not yet been fixed are highlighted in pink.

Definitions Tab

This section lists all the alerts (single resource alerts, resource-type alerts, and group alerts) defined across the HQ deployment, regardless of whether it has been triggered or not, sorted by creation date and time, in backwards chronological order. In the "Definition Filter" section, users can filter the list of alert definitions displayed at the right.

For each listed alert definition, the screen lists:

  • Name: The alert definition's name
  • Created: Date and time the alert was defined
  • Modified: Date and time the alert was modified (if at all)
  • Active: Whether or not the alert definition is configured to generate alerts) or not. Starting in HQ 4.1.1, if the alert is defined to fire once, and not again until after it is fixed, an yellow flag icon will be displayed in this column if the the alert is not fixed.
  • Last Alert: Date and time the alert was last triggered
  • Resource or Resource Type or Group: The resource, resource type, or group for which the alert was defined, respectively
  • Escalation: The name of the escalation scheme assigned to the alert definition
  • Priority: The alert definition's priority

Navigating via the Masthead Menu

Main screens in Hyperic contain a masthead menu. Users can navigate to the following parts of HQ using this menu:

Dashboard Displays the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Resources Click the menu name to access either Browse (takes users to the Browse Resources screen, where they can locate and navigate through information about all managed resources), Currently Down (takes users to the Currently Down screen, where they can look at all managed resources that are unavailable), Nagios Availability (available only when Nagios is installed), or Recently Viewed (provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the Current Health screen for the resource).
Analyze Click the menu name to access Reporting * (presents the Reporting screen, from which reports can be generated), Alert Center (presents the Alert Center, which displays a deployment wide alert summary), or Event Center (presents the Event Center, which displays a deployment-wide event summary).
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the Administration screen, where they can manage the system-wide settings
Search
The Search box in the upper right of the Masthead allows you to search for HQ resources of any type and HQ users.

Search results appear after four characters are entered in the text box. Results will include the first 10 platforms, servers, services, and groups whose name includes the search string.

Double-click a resource in the list to navigate to its Resource page.

The masthead also displays the two most recently triggered alerts. Users can click the alert's time to be taken to the alert detail.
