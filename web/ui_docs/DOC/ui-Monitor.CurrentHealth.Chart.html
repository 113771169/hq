Tasks Available on This Screen

On this screen, users can do the following tasks:

To change the baseline or expected range of values for the charted metric:

  1. In "Metric Baseline & Expected Range," click Change Value next to the Baseline, High Range, or Low Range.
  2. Either leave the displayed calculated value or enter another value.
    You must enter the unit of measure appropriate for the selected metric. For example, % for the Availability metric or MB for the Free Memory metric.
    If specifying both high and low values, the low must be lower than the high.
  3. Click Save Value.
    HQ automatically checks "Baseline," "High Range," or "Low Range" (depending on what was changed) in the chart legend and refreshes the chart with those values.

Participating Resource

This section lists the resource or resources (in the case of charting a group) and their metrics that are plotted in the chart. Users can choose to have some or all of the charts redrawn.

Metric Baseline and Expected Range

This section allows the user to view and change the values current calculated or set for the metric baseline and the expected range of metric values. This section is available only when a single metric is charted.

Reading a Chart

The Y-axis represents the metric value; the X-axis, time. Each column represents a 1/60 time slice of the selected metric display range. A chart can plot a maximum of 60 points. The chart displays:

  • The average, peak, and low observed values for the metric for the slice of time along the X-axis
  • If available, the established baseline, high range, and low range values

Each column displays :

  • The actual observed metric value for that time. For dynamic metrics, this can be itself an average. See details below.
  • The range (represented by the I bar) of metric values collected in the preceding time slice

Display of Dynamic Data vs. Static Data

For static metrics, the plotted point represents a single metric value at one moment in time. For dynamic metrics, a point on the chart represents an average metric value for a period of time that ended at the corresponding X-axis value. How long is that period of time over which the value was averaged? That depends on the display range. For a display range of eight hours, each charted point represents the average of the preceding eight-minute period (8 hours / 60 points along the X-axis = 8 minutes). If the metric is collected every 60 seconds, and the chart's display range is 60 minutes, each of the 60 plotted points on the graph represents the single, observed value for the metric at a single point in time.

Display of Cumulative Data

The values for data that is cumulative (trending up or down) — such as bytes served, uptime, minimum response time, number of transactions — are not averaged. A point charted for this kind of data shows the maximum (or minimum) value for the period of time the point represents.

Navigating via the Masthead Menu

Main screens in Hyperic contain the same menu in the masthead. Users can navigate to the following parts of HQ using this menu:

Dashboard Takes users to the starting screen in HQ, which displays information about resource health, recent alerts, recently performed Auto-Discoveries, and control actions.
Browse Resources Takes users to the screen where they can locate and navigate through information about all managed resources
Administration Takes users (who are assigned to a role that has the Administer Hyperic HQ Server Configuration option enabled) to the screen where they can manage the Hyperic HQ Server configuration.
Recently Viewed Resources Provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the inventory of that resource.
Help Opens screen-sensitive online help in a secondary window
