HQ Dashboard

Topics marked with * relate to HQ Enterprise-only features.

This screen is a user's starting point in HQ. Users can organize the screen to correspond to their workflow or infrastructure. By default, the screen displays information about resource health, recent alerts, recently performed auto-discovery scans, and control actions. In addition, users can search for specific resources and view summary counts for all managed resources.

The Dashboard is simply a compendium of user-selected portlets. Each portlet contains a different type of data. Users can manipulate which portlets are displayed, where they are displayed, and what data they display.

The data (portlets) displayed in the Dashboard is determined by the user's role(s) via role-based Dashboards. *

Tasks Available on This Screen

Users can customize the Dashboard so that, when they log in, it displays only the information that matters to them. Portlets can be added and removed from the Dashboard, and users can view and manage each portlet's configuration information (the amount or kind of data that is displayed in the portlet and how it is displayed). Portlets can be moved around the screen to make the display most useful. Also, some portlets can update users through RSS.

For all portlet-specific tasks, see the description of the portlets below.

To add a portlet:

  • In Add content to this column at the bottom of either the right or left side of the screen, select the portlet name and click .

To delete a portlet:

  • Click in the upper right-hand corner of the portlet.

To move a portlet's position on the screen:

  • Click the header of the portlet and drag and drop it to its desired location with its column.

To change the configuration of the portlet:

  • Click in the upper right-hand corner of the portlet.

To get the RSS-feed URL from RSS-enabled portlets:

  • Click in the upper left-hand corner of the portlet or copy the icon's link.

To view a different Dashboard: *

  • Select a different role-based Dashboard in Select a Dashboard in the upper left.
    Only those Dashboards associated with the roles the user is assigned to are available for selection.

Available Portlets

Users can select from among the following portlets on the Dashboard (listed in alphabetical order):

Portlet Name Portlet Description
Auto-Discovery Lists the platforms and their hosted servers that HQ has most recently discovered via auto-discovery. These resources now need to be added to the HQ inventory. When a platform is added, all its hosted servers are, by default, also added; however, users can opt to add a platform without its servers. When a server is added, its hosting platform is always added. (Services are automatically added to the HQ inventory as part of their hosting platforms or servers.) By default, results of only the last five auto-discovery scans are shown. "Skipping resources": Sometimes HQ discovers a resource (specifically here, a server) that users just don't care about. If that is the case with a server displayed here, users can banish the server from display in the portlet by "skipping" it. For a skipped server to be displayed in the UI again, it must first be rediscovered in a new auto-discovery scan and then be marked for "Import" on the "Auto-Discovery Results" screen.
  • To change the number of auto-discovery scan whose results are displayed in the portlet:
    • Click and proceed to the "HQ Dashboard Settings: Auto-Discovery Queue" screen.
  • To add a discovered platform to the HQ inventory:
    1. Select the platform's radio button.
    2. To not add some of the platform's hosted servers, uncheck those servers.
    3. Click Add to inventory.
  • To add a discovered server to the HQ inventory:
    • Check the server and click Add to inventory.
  • To skip a server:
    • Check the server(s) and click Skip Checked Resources
  • To view the results of an auto-discovery scan on a listed platform:
    • Click the platform's name and proceed to the "Auto-Discovery Results" screen.
Availability Summary Displays the availability of a user-selected set of resources, aggregated to the resource type-level. This portlet refreshes every minute.
  • To change the set of resources counted in this portlet:
    • Click and proceed to the "HQ Dashboard Settings: Availability Summary" screen.
Control Actions Lists the control actions that HQ has most recently performed on any of the managed resources (either as part of an alert action or by schedule) ("Recent Control Actions"), upcoming scheduled control actions ("Scheduled Control Actions"), and the most frequently performed "quick control" actions ("Quick Control Frequency").
  • To change the number of control actions displayed in the portlet:
    • Click and proceed to the "HQ Dashboard Settings: Control Actions" screen.
  • To view the control-action history of a listed resource:
    • Click the resource name and proceed to the "Control History"" screen.
Favorite Resources Lists user-selected favorite resources. Resources can be designated as "favorite" throughout the UI via the Add to Dashboard Favorites link in the Tools Menu (which appears on most monitoring and inventory screens). This portlet refreshes every minute. The portlet shows the resource's:
  • Performance. If a single performance metric is being collected for the resource, that value is displayed here. If several performance metrics are being collected, then HQ chooses the one marked as an indicator metric. An "N/A" value indicates that no performance metrics are being collected for the resource. To see what performance metrics are being collected for a resource, click the resource name and proceed to the "Current Health" screen. Click . (Help for that tab)
  • Availability
  • Number of triggered alerts
    To add resources to this portlet:
    • Click and proceed to the "HQ Dashboard Settings: Favorite Resources" screen.
Group Alert Summary* Displays alert status for the groups in the the portlet. Two traffic light indicators are shown for each group. The Resource Alerts traffic light indicates the status of alerts on individual group members. The Group Alerts traffic light indicates the status alerts on the group as a whole. (A group alert fires if its condition is "true" for a percentage, or a specific number, of group members). 
Click on a traffic light for a group alert to view a list of alerts that have fired.
Click on a group name to view the group page.
  • Green - Indicates no that there are no fired alerts, or that all fired alerts have been fixed.
  • Yellow - Indicates that there is one or more unfixed alerts in escalation that have been acknowledged, and are currently "paused".  (An escalation can be configured to allow a user to temporarily pause the escalation process when acknowledging the alert.)  
  • Red - Indicates unfixed alerts.
  • Grey - A grey traffic light is shown if there are no active alerts of that type.

    To add groups to the portlet, click . Move desired groups from the Available groups list to the Enabled groups list and click save. To filter the Available groups list, enter a search string in the search box.
Hyperic IQ Reports* In HQ Enterprise 4.1 and later, in deployments that include HQ Hyperic IQ, this portlet allow users to run IQ reports that reside in Hyperic IQ's Dashboard Reports directory. The HQ Server must be configured with the Hyperic IQ URL. For more information see Edit HQ Server Settings.
Metric Viewer Shows the user-selected metrics for a user-selected set of resources. This portlet refreshes every minute.
  • To change the metrics or resources in this portlet:
    • Click and proceed to the "HQ Dashboard Settings: Metric Viewer" screen.
Problem Resources* Lists resources that have recently had Problem Metrics. You can configure look-back period, up to maximum of 48 hours, and the number of resource to list, up to 30. Resources are ordered first by inventory level - platforms, then servers, then services. Within an inventory level, resources are ordered by the date/time of most recent problem, starting with most recent.  For each resource, availability, the number of fired alerts, the number of out-of-bound (OOB) metrics reported, and the timestamp of the most recent problem are shown. The data refreshes every minute. 
  • To change how many hours to consider and the number of resources to include:
    • Click and proceed to the "HQ Dashboard Settings: Problem Resources" screen.
  • To view metrics for a problem resource:
    • Click the resource name to display the resource's "Current Health" page.
Recent Alerts Lists the most recently triggered alerts for a set of user-selected resources. This portlet refreshes every minute.
  • To select the resources whose alerts are displayed in the portlet or to change the number of alerts displayed:
    • Click and proceed to the "HQ Dashboard Settings: Alerts" screen.
  • To mark a single alert or group of alerts fixed:
    • Check the alert(s) and click Fixed.
    • A pop-up box will prompt you to enter a comment describing the fix. As of HQ 4.1.1, if there are other unfixed alerts with the same definition, you are offered the option to "Fix all previous alerts".
  • To acknowledge an alert:
    • Click in the alert's row. As of HQ 4.1.1, if the escalation has pause options, you are offered the option to pause the escalation.
      The icon is available only for as-yet-unacknowledged alerts with escalation schemes assigned to them.
Recently Added Lists all platforms recently added to the HQ inventory and how long ago it was added. By default only platforms added in the last 48 hours are listed.
  • To change the number of resources displayed in the portlet:
    • Click and proceed to the "HQ Dashboard Settings: Recently Added" screen.
  • To view a recently added resource:
    • Click the alert name and proceed to the "Current Health" screen for the resource.
Search Resources Allows users to search for resources in the HQ inventory. The search supports case-insensitive, partial-term queries and is restricted to a single inventory level per search.
  • To search for resources:
    • Type in at least part of the resource name, select the resource's inventory level and click .
      HQ displays all matching resources on the "Browse Resources" screen.
Summary Counts Displays a count of all the managed resources of each inventory level (how many platforms, services, etc.) that the user has the permissions to view. HQ includes all resource types in the counts, but users can choose to see additional counts of specific resource types. Users can also create new applications, groups, and platform.
  • To view counts of a subset of resource types:
    • Click and proceed to the "HQ Dashboard Settings: Summary Counts" screen.
  • To view all the resources of a particular inventory level:
    • Click inventory level or the count.
      HQ displays all the resources of that level in the "Browse Resources" screen.
  • To create a new application:
    • Click New Application and proceed to the "New Application" screen.
  • To create a new group:
    • Click New Group and proceed to the "New Group" screen.
  • To create a new platform:
    • Click New Platform and proceed to the "New Platform" screen.
Saved Charts*  Displays saved charts as a slideshow. Charts can be saved to the Dashboard from the "Charts" screen.

Click to display the Saved Charts Configuration panel.  You can toggle Chart Rotation on and off, set the Rotation Interval and the Time Range over which the data is charted. 

Chart rotation can be toggled on and off.

Navigating via the Masthead Menu

Main screens in Hyperic contain a masthead menu. Users can navigate to the following parts of HQ using this menu:

Dashboard Displays the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Resources Click the menu name to access either Browse (takes users to the Browse Resources screen, where they can locate and navigate through information about all managed resources), Currently Down (takes users to the Currently Down screen, where they can look at all managed resources that are unavailable), Nagios Availability (available only when Nagios is installed), or Recently Viewed (provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the Current Health screen for the resource).
Analyze Click the menu name to access Reporting * (presents the Reporting screen, from which reports can be generated), Alert Center (presents the Alert Center, which displays a deployment wide alert summary), or Event Center (presents the Event Center, which displays a deployment-wide event summary).
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the Administration screen, where they can manage the system-wide settings
Search
The Search box in the upper right of the Masthead allows you to search for HQ resources of any type and HQ users.

Search results appear after four characters are entered in the text box. Results will include the first 10 platforms, servers, services, and groups whose name includes the search string.

Double-click a resource in the list to navigate to its Resource page.

The masthead also displays the two most recently triggered alerts. Users can click the alert's time to be taken to the alert detail.
