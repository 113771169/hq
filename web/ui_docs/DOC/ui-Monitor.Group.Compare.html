Group Compare Metrics

Available in HQ Open Source unless marked by * for HQ Enterprise only

This screen displays all available metric data, in a format conducive to comparison across resources, for selected group members. If a resource does not have data for a listed metric, the resource is simply not listed with the metric. This screen does not automatically refresh and therefore must be manually refreshed.

Tasks Available on This Screen

On this screen, users can do the following:

To chart a metric for the listed resources:

  • Click the metric name and proceed to the "Metric Chart" screen.

To refresh the displayed metric values:

  • Click next to Get Current Values at bottom-right.

To change the metric display range:

  • Follow the instructions for doing the same thing on the "Current Health" screen.

Sections on This Screen

This screen organizes the metrics by category, which are explained here. For each metric, for each resource being compared, the screen displays the:

  • LOW: The lowest observed metric value
  • AVG: The average of the observed metric values
  • PEAK: The highest observed metric value
  • LAST: The last observed metric value
Metrics with "--" Low, Avg, and Peak Values

HQ displays dashes for the Low, Avg, and Peak of throughput metrics because the values are ever-increasing. The screen will indicate a "per <time value>," and the associated rate is calculated by HQ using the cumulative throughtput data that the HQ plugin elicits from the resource. This value is displayed as Last. For example, for Apache, HQ cannot get "requests served per minute" data. It instead gets "total requests served" and calculates and displays how many requests are added per minute on top of the Apache counter.
