HQ Administration

Available in HQ Open Source unless marked by * for HQ Enterprise only

This screen allows Administrators to configure their deployment of Hyperic HQ: manage users and roles, monitoring (metrics) defaults and alert-definition defaults (including escalation schemes) for each resource type, manage the HQ Server configuration, and view license information for this deployment. This screen and all its component functions are available to Administrators only.

Tasks Available on This Screen

On this screen, users can do the following tasks:

To view a list of all users:

  • Click List Users and proceed to the "List Users" screen.

To create a new user:

  • Click New User and proceed to the "New User" screen.

To view a list of all roles: *

  • Click List Roles and proceed to the "List Roles" screen.

To create a new role: *

  • Click New Role and proceed to the "New Role" screen.

To manage the HQ server:

  • Click HQ Server Settings and proceed to the "Edit HQ Server Settings" screen.

To manage monitoring and alerting defaults:

  • Click Monitoring Defaults and proceed to the "HQ Monitoring Defaults Configuration" screen.

To manage escalation schemes for use in alerts:

  • Click Escalation Schemes Configuration and proceed to the "Escalation Schemes Configuration" screen.

To access the HQ Health plugin diagnostics:

  • Click HQ Health in the "Plugins" section and proceed to the "HQ Health" screen.

To access the Groovy console:

  • Click Groovy Console in the "Plugins" section and proceed to the "Groovy Console" screen.

Sections on This Screen

Authentication/Authorization

This section gives users access to all the users and roles defined in the system and allows them to create new ones.

HQ Server Settings

"HQ Server Configuration" allows users to view the current configuration settings for the Hyperic HQ server. "Monitoring Defaults Configuration" allows viewers to view current monitoring defaults for each resource type. And in "Escalation Schemes Configuration," users can define escalation schemes for use in alerts throughout HQ.

Plugins

This section lists all the UI plugins — standard or custom — that are installed on HQ. The two standard plugins are:

  • HQ Health: This plugin provides real-time diagnostic data for the machine on which the HQ Server is installed. The data is not intended for customers to use on their own, but instead to help HQ Support troubleshoot problems with a customer's deployment.
  • Groovy Console: This plugin allows users to run Groovy code directly in the HQ Server.

To access either feature, click the plugin name.

HQ License Information*

This section lists the license information for the HQ server:

  • Name: The name on the Hyperic account
  • Expires: The date the license expires
  • Platform Limit: The number of platforms (network boxes) the license allows HQ to manage
  • Current Platform Count: The number of those platforms that are currently being managed

Navigating via the Masthead Menu

Main screens in Hyperic contain a masthead menu. Users can navigate to the following parts of HQ using this menu:

Dashboard Takes users to the Dashboard, the starting screen in HQ, which displays information about resource health, recent alerts, recently performed auto-discovery scan, and control actions
Resources Click the menu name to access either Browse (takes users to the "Browse Resources" screen, where they can locate and navigate through information about all managed resources), Currently Down (takes users to the "Currently Down" screen, where they can look at all managed resources that are unavailable), Nagios Availability (available only when Nagios is installed), or Recently Viewed (provides a drop-down list of resources that the user has recently viewed. Selecting one of those resources takes the user to the "Current Health" screen for the resource).
Analyze Click the menu name to access either Reporting * (takes users to the "Reporting" screen, where they can generate all reports available in HQ), Alert Center (takes users to the Alert Center, where they can look at all alerts triggered across the system).
Administration Takes users (with "Administer Hyperic HQ Server Configuration" permissions) to the "Administration" screen, where they can manage the system-wide settings

The masthead also displays the two most recently triggered alerts. Users can click the alert's time to be taken to the alert detail.
